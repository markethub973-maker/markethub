# Public HTTP Status Audit — 2026-04-16

Comprehensive HTTP audit of all public-facing routes on
`markethubpromo.com` and `brain.markethubpromo.com`.

## Method
- URL list sourced canonically from:
  - `src/proxy.ts` → `PUBLIC_PATHS` array
  - `src/lib/featuresData.ts` → 18 feature slugs → `/features/<slug>`
  - `src/lib/guidesData.ts` → 9 guide slugs → `/guides/<slug>`
  - Plus: offer landers (RO/INTL/DE/FR/IT/ES/PT), `/offer/thanks`,
    `/promo`, `/pricing`, `/features`, `/guides`, `/help`,
    `/changelog`, `/status`, `/security`, `/blocked`, `/login`,
    `/register`, `/privacy`, `/terms`, `/upgrade-required`,
    `/api/health`, `/api/status`, `/api/docs`, `/api/pricing`,
    `/sitemap.xml`, `/robots.txt`, `/.well-known/security.txt`,
    `brain.markethubpromo.com/` + `/login`
- Probe: `curl -s -o /dev/null -w "%{http_code}|%{time_total}"`,
  `--max-time 20`, User-Agent `AuditBot/1.0`, 10-way parallel via
  `xargs -P 10`. No authenticated cookies, no follow-redirects.

## Results
- **Total URLs tested:** 58
- **OK (200):** 58
- **Broken (4xx/5xx unexpected):** 0
- **Slow (>3s):** 0
- **Unexpected redirects:** 0

## Broken URLs
_None._ All 58 probed URLs returned `200 OK` on first hop.

## Slow URLs (>3s)
_None._ Slowest responses observed:
- `/api/status` — 0.909s
- `/api/health` — 0.782s
- `/features` — 0.664s
- `/offer-es` — 0.646s
- `/api/pricing` — 0.637s

Median response ≈ 0.44s. All well under the 3s alert threshold.

## Security-Relevant Findings
- `brain.markethubpromo.com/` returned **200** to an unauthenticated
  probe. This is **expected and correct**: `proxy.ts` lines 310-313
  rewrite any unauthenticated request on this host to the internal
  `/brain-login` page (rewrite, not redirect — hence 200 with a login
  form body). The actual admin dashboard is NOT served without the
  `brain_admin=1` cookie. No leak.
- `brain.markethubpromo.com/login` returned 200 (login page render) —
  same mechanism, expected.
- `/.well-known/security.txt` returned 200 — RFC 9116 compliance
  verified.
- `/sitemap.xml` + `/robots.txt` returned 200 — SEO crawlers
  unblocked, as required by `PUBLIC_PATHS`.
- `/api/health`, `/api/status`, `/api/docs`, `/api/pricing` returned
  200 unauthenticated — all four are **explicitly public** per
  `PUBLIC_PATHS` comments (uptime probes, status feed, public API
  docs, pricing page used pre-login on `/register`).
- No route that should require auth returned 200.

## Conclusion
Public surface is fully healthy. No 404/500, no slow pages, no
accidental exposure of authenticated routes. Admin tunnel
(`/markethub973`, `/dashboard/admin`, `/api/admin/*`) was NOT probed
in this audit because those paths intentionally return 404 without
`ADMIN_TUNNEL_SECRET`; they are not in scope of a "public pages"
health check.
